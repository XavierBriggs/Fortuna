## Findings

- [Major] example.toml placement of max_book_age_ms comment is at TOML root (line 10, before any section header), not under [gates]. When operator uncomments the line, serde silently ignores it (FortunaConfig has no deny_unknown_fields) and gates.max_book_age_ms remains None — the book-age check is silently disabled, contradicting operator intent. Reproduction: `python3 -c "import tomllib; print(tomllib.loads('max_book_age_ms = 5000\n\n[gates.global]\nmax_total_exposure_cents = 1').get('gates', {}).get('max_book_age_ms'))"` → None. Fix: place under an explicit `[gates]` header or use `gates.max_book_age_ms = 5_000` inline.

- [Minor] example.toml comment says "Values <= 0 are accepted but trivially reject all orders backed by a book" — inaccurate for max=0: a book with age_ms=0 (as_of == now) passes (0 > 0 = false). True for max < 0. Not a code defect.
