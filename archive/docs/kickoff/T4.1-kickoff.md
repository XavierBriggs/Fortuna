  mind_from_env returns the paid mind — a test that sources .env spends
  real money and breaks determinism. Tests inject StubMind/mock
  transports and synthetic env maps; only the OPERATOR runs the binary
  against the real env.
- Do NOT ping FORTUNA_DEADMAN_URL from any test or smoke — the first
  ping ARMS the external monitor and silence afterwards pages the
  operator. The pinger activates only in the real daemon run; say so in
  the runbook output.
- Kill-switch crate independence (I4): fortuna-killswitch must gain ZERO
  new deps from this work. Don't touch crates/fortuna-invariants/ at all
  (any touch joins the operator waive queue).
- Money paths: no unwrap/expect/panic (anyhow is fine in the BINARY
  only). Integer cents everywhere; probabilities f64 in cognition only.
- Secrets: never in logs, audit payloads, or Debug output (transports
  already redact; keep it that way in the daemon's startup logging).
- Slack: every outbound message ALSO writes an audit row (spec 8).
  SlackRouter is fail-closed at construction — boot must surface that,
  not panic.
